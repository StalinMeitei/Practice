# Private Key Troubleshooting Guide

## The Problem

You're seeing this error in django-pyas2 admin:
```
Invalid Private Key or password is not correct.
```

## The Solution

**LEAVE THE PASSWORD FIELD EMPTY!**

The private keys generated by `generate_certificates.py` are **NOT password protected**. Django-pyas2 is asking for a password field, but you should leave it blank.

## Step-by-Step Fix

### 1. Verify Your Key Format

Run the checker:
```bash
python fix_private_key_format.py
```

Your key should show:
```
✓ Key format is correct
```

### 2. Upload Key in Django Admin

1. Go to: http://localhost:8001/admin/
2. Click: **Organizations**
3. Click on your organization (e.g., "P1")
4. Scroll to **Private Key** section
5. Click **Choose File** next to "Key file"
6. Select: `P1_private.pem`
7. **IMPORTANT**: Leave "Private Key Password" field **EMPTY**
8. Click **Save**

### 3. What NOT to Do

❌ Do NOT enter any password
❌ Do NOT enter "password", "admin", or any other text
❌ Do NOT use encrypted keys
❌ Do NOT use .key or .crt files separately

### 4. What TO Do

✅ Use the `_private.pem` file (e.g., P1_private.pem)
✅ Leave password field completely empty
✅ Make sure the file contains both private key AND certificate
✅ Use PKCS#8 format (not RSA/PKCS#1)

## Understanding the Key Format

### Correct Format (PKCS#8 + Certificate)

Your `P1_private.pem` should look like this:

```
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQDA0+VeECIg1zL2
[... more lines ...]
-----END PRIVATE KEY-----
-----BEGIN CERTIFICATE-----
MIIDezCCAmOgAwIBAgIUGHrTFEmzjVnavIiNPe6fYLpJYq4wDQYJKoZIhvcNAQEL
[... more lines ...]
-----END CERTIFICATE-----
```

### Key Points:

1. **Header**: `-----BEGIN PRIVATE KEY-----` (NOT "RSA PRIVATE KEY")
2. **No Encryption**: No "ENCRYPTED" text anywhere
3. **Certificate Included**: Contains both private key and certificate
4. **Format**: PKCS#8 (modern standard)

## Common Issues and Fixes

### Issue 1: "Invalid Private Key" Error

**Cause**: Password field is not empty

**Fix**: 
1. Clear the password field completely
2. Make sure there are no spaces
3. Save again

### Issue 2: Wrong Key Format

**Cause**: Using PKCS#1 (RSA) format instead of PKCS#8

**Fix**: Regenerate certificates
```bash
python generate_certificates.py
```

### Issue 3: Encrypted Key

**Cause**: Key file contains "ENCRYPTED" in header

**Fix**: Generate new unencrypted keys
```bash
python generate_certificates.py
```

### Issue 4: Separate Key and Certificate Files

**Cause**: Using separate .key and .crt files

**Fix**: Use the combined `_private.pem` file that contains both

### Issue 5: Wrong File Selected

**Cause**: Selected `_public.pem` instead of `_private.pem`

**Fix**: 
- For P1: Use `P1_private.pem` (not P1_public.pem)
- For P2: Use `P2_private.pem` (not P2_public.pem)

## Verification Steps

### 1. Check File Contents

```bash
# Windows PowerShell
Get-Content P1_private.pem | Select-Object -First 5

# Should show:
# -----BEGIN PRIVATE KEY-----
# MIIEvQIBADANBgkqhkiG9w0BAQEFAASCBKcwggSjAgEAAoIBAQDA0+VeECIg1zL2
# ...
```

### 2. Check for Encryption

```bash
# Windows PowerShell
Get-Content P1_private.pem | Select-String "ENCRYPTED"

# Should return nothing (no matches)
```

### 3. Check for Certificate

```bash
# Windows PowerShell
Get-Content P1_private.pem | Select-String "BEGIN CERTIFICATE"

# Should show:
# -----BEGIN CERTIFICATE-----
```

## Regenerating Certificates

If your keys are in the wrong format, regenerate them:

```bash
# Backup old keys
copy P1_private.pem P1_private.pem.backup
copy P2_private.pem P2_private.pem.backup

# Generate new keys
python generate_certificates.py

# Verify format
python fix_private_key_format.py
```

## Why This Happens

Django-pyas2 supports both encrypted and unencrypted private keys:
- **Encrypted keys**: Require a password
- **Unencrypted keys**: Password field must be empty

Our `generate_certificates.py` creates **unencrypted keys** for ease of use in Docker environments, so the password field must remain empty.

## Alternative: Using Encrypted Keys

If you want to use password-protected keys:

1. Generate encrypted key:
```python
from cryptography.hazmat.primitives import serialization

# When saving private key:
private_key.private_bytes(
    encoding=serialization.Encoding.PEM,
    format=serialization.PrivateFormat.PKCS8,
    encryption_algorithm=serialization.BestAvailableEncryption(b'your_password')
)
```

2. In Django admin:
   - Upload the encrypted key file
   - Enter the password in "Private Key Password" field

## Quick Reference

| Scenario | Password Field |
|----------|---------------|
| Unencrypted key (default) | **EMPTY** |
| Encrypted key | Enter password |
| No key uploaded | N/A |

## Summary

✅ **DO THIS**:
- Use `P1_private.pem` or `P2_private.pem`
- Leave password field **EMPTY**
- Make sure file contains both key and certificate
- Use PKCS#8 format

❌ **DON'T DO THIS**:
- Enter any password
- Use separate .key/.crt files
- Use encrypted keys without password
- Use PKCS#1/RSA format keys

---

**The most common mistake**: Entering something in the password field when the key is not encrypted. Just leave it empty!
